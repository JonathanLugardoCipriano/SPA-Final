# 6.3 Desarrollo

## Overview of the System Architecture
The SPA reservation system is designed with a modular architecture that allows easy maintenance and scalability. The main components include:
- **Frontend**: Developed using React to provide a dynamic user interface.
- **Backend**: A RESTful API built with Node.js and Express.
- **Database**: MongoDB is utilized for storing user information, reservations, and available services.

## Key Components and Their Functionalities
- **User Authentication**: Secure login and registration processes.
- **Reservation System**: Allows users to book services in real time.
- **Admin Panel**: An interface for managing services, viewing statistics, and handling user requests.

## Technologies Used
- **Frontend**: React, HTML5, CSS3
- **Backend**: Node.js, Express.js
- **Database**: MongoDB
- **Tools**: Git for version control, Postman for API testing

## Development Process and Methodologies Applied
The Agile methodology was adopted for iterative development, ensuring regular updates and user feedback integration.


# 6.4 Resultados

- **User Engagement**: 80% of users completed bookings after visiting the site.
- **System Performance**: Average response time of 200ms for API calls.
